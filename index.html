Prokaryotic Pangenomes as Bet-Hedging Devices

A theoretical and empirical framework showing that accessory gene repertoires in prokaryotes function as evolved bet-hedging strategies under environmental uncertainty.

About the paper

Prokaryotic pangenomes — the full complement of genes found across a species — are typically partitioned into a conserved “core” and a variable “accessory” genome. The accessory genome can comprise the majority of a species’ gene repertoire, yet the evolutionary logic governing its composition has remained unclear.

This work presents a unifying explanation: pangenome structure is a bet-hedging device. Through mathematical modelling and analysis of >10,000 E. coli genomes and 670 prokaryotic species, I show that maintaining within-population gene-content diversity acts as a form of biological insurance. Lineages that spread their genetic “investments” across environments via accessory gene portfolios reduce long-term fitness variance, analogous to portfolio diversification in financial theory.

Horizontal gene transfer (HGT) emerges as the mechanism that sustains these diversified portfolios at equilibrium, and the characteristic U-shaped gene-frequency distribution observed across pangenomes arises naturally from the model without additional assumptions.

This repository contains all simulation and analysis code required to reproduce every figure in the manuscript.

Quick start

Clone the repository, install dependencies, download the data, and generate figures:

# Clone and set up
git clone https://github.com/mol-evol/pangenome-bet-hedging.git
cd pangenome-bet-hedging
pip install -r requirements.txt

# Download the Horesh et al. dataset (~790 MB)
cd data && bash download_data.sh && cd ..

# Generate all figures
python run_figures.py --all

# Or generate specific figures
python run_figures.py --figures 1 3        # main figures 1 and 3
python run_figures.py --supplementary S1 S2 # supplementary figures
python run_figures.py --figures 4 5          # empirical figures
python run_figures.py --quick                # fast mode (fewer replicates)
python run_figures.py --dry-run              # preview commands only

Output is saved to output/ by default. Override with --output-dir PATH.

Running tests

python -m pytest empirical/test_*.py -v
